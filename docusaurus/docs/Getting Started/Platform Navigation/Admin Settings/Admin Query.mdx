The Admin Query allows administrators to use SQL queries to update the platform's databases. This can ensure the platform aligns with organizational branding, data and a consistent user experience.

## Navigating to the Admin Query
Only admin users can modify the theme settings. 

To access the Admin Query:
1. Open up the side menu. Click on Settings.


![SettingsButton](../../../../static/autoImg/PlatformNavigation/Admin%20Query/Settings.png)




2. Once on the Settings page, enable the Admin mode. 


![Enable Admin Mode](../../../../static/autoImg/PlatformNavigation/Admin%20Query/Admin%20On.png)


3. Click on the **Admin Query** tile.


![AdminTile](../../../../static/autoImg/PlatformNavigation/Admin%20Query/Admin%20Query.png)

## Local Master Database
Admin query allows you to track and edit the user credentials, applications, projects, engines and functions within the AI Core platform. 
### Track Engines
1. To run the Admin Query and to see available engines, use the following SQL command as an example:
```sql
SELECT * FROM Engine
``` 
2. Select **LocalMasterDatabase** from the Database dropdown.
![LocalMasterDropdown](../../../../static/autoImg/PlatformNavigation/Admin%20Query/LocalMasterDatabase.png)

3. Paste the query into the text area and click **Run**.
![ExecuteLocalMaster](../../../../static/autoImg/PlatformNavigation/Admin%20Query/LocalMasterDatabaseQuery%20Results.png)

## Security Database
Admin query lets you manage members and their permissions within the platform.
### Track Engine Permissions
1. To run the Admin Query and to see engine permissions, use the following SQL command as an example:
```sql
SELECT * from ENGINEPERMISSION
``` 
2. Select **security** from the Database dropdown.
![SecurityDropdown](../../../../static/autoImg/PlatformNavigation/Admin%20Query/security.png)

3. Paste the query into the text area and click **Run**.
![ExecuteSecurity](../../../../static/autoImg/PlatformNavigation/Admin%20Query/securityQuery%20Results.png)


## Scheduling Database
Admin query allows you to track and manage automated jobs within the platform.
### Track Jobs
1. To run the Admin Query and to see job details, use the following SQL command as an example:
```sql
SELECT * FROM QRTZ_JOB_DETAILS
``` 
2. Select **scheduler** from the Database dropdown.
![SchedulerDropdown](../../../../static/autoImg/PlatformNavigation/Admin%20Query/scheduler.png)

3. Paste the query into the text area and click **Run**.
![ExecuteScheduler](../../../../static/autoImg/PlatformNavigation/Admin%20Query/schedulerQuery%20Results.png)

### Important Theme Values
Below is an explanation of each important key used in the Admin Query:

- **termsHeaderReact**
  - **Description**: This key sets the header text for the terms and conditions section.
  - **Key**: `"termsHeaderReact":"Attention"`
  - **Purpose**: Displays the header "Attention" in the terms and conditions section.

- **termsReact**
  - **Description**: This key contains the HTML content for the terms and conditions.
  - **Key**: `"termsReact": "Please adhere to the member firm and geography GenAI guardrails and consult your risk team for your specific use case & data usage..."`
  - **Purpose**: Provides detailed terms and conditions for platform users.

- **cookiePolicyBannerReact**
  - **Description**: This key holds the HTML content for the cookie policy banner.
  - **Key**: `"cookiePolicyBannerReact":"This site uses strictly necessary cookies and similar technologies to operate this website and to provide you with a more personalized user experience..."`
  - **Purpose**: Displays a banner informing users about the use of cookies and links to the cookie policy.

- **cookiePolicyOrderReact**
  - **Description**: This key specifies the order of cookie policies.
  - **Key**: `"cookiePolicyOrderReact":["Strictly Necessary Cookies"]`
  - **Purpose**: Determines the sequence in which cookie policies are presented to users.

- **cookiePoliciesReact**
  - **Description**: This key contains the descriptions of various cookie policies.
  - **Key**: `"cookiePoliciesReact": {"Strictly Necessary Cookies": "These cookies are essential for you to browse the website and use its features, such as accessing secure areas..."}`
  - **Purpose**: Provides detailed descriptions of each cookie policy type.

- **cookiePolicyNoticePage**
  - **Description**: This key holds the HTML content for the cookie policy notice page.
  - **Key**: `"cookiePolicyNoticePage":"Last revised: 12/2/24Cookie NoticeWhat is a cookie?A cookie is a small text file placed on your device by the websites that you visit..."`
  - **Purpose**: Displays the full cookie policy notice to users.

- **privacyNoticePage**
  - **Description**: This key contains the HTML content for the privacy notice page.
  - **Key**: `"privacyNoticePage":"AI Core Platform Privacy NoticeLast revised: 12/2/24This notice covers your use of this system and the limited processing of personal information..."`
  - **Purpose**: Provides users with detailed information about the platform's privacy practices.

- **helpBannerOrder**
  - **Description**: This key specifies the order of items in the help banner.
  - **Key**: `"helpBannerOrder":["documentation","tutorials","privacy-notice","platform-tour","github","feedback"]`
  - **Purpose**: Determines the sequence in which help banner items are displayed.

- **helpBannerValues**
  - **Description**: This key contains the labels and URLs for the help banner items.
